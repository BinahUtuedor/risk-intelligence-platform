# Model Documentation – Risk Intelligence Platform

## Overview

The Risk Intelligence Platform uses a supervised machine learning approach to classify countries into risk categories based on economic, governance, regulatory, and compliance indicators.

The objective of the model is to identify countries that may require additional due diligence, compliance review, or investigation.

The model predicts:

* **High Risk (1)**
* **Low Risk (0)**

The resulting predictions support risk-based decision-making within compliance, AML, KYC, and financial crime operations.

---

# Model Objective

The model is designed to answer the following business question:

> Which countries present elevated compliance and operational risk based on economic conditions, governance quality, sanctions exposure, and regulatory indicators?

The model converts multiple risk indicators into a single probability score and classification output.

---

# Machine Learning Algorithm

## Random Forest Classifier

The platform uses a Random Forest classification model from Scikit-Learn.

Implementation:

```python
from sklearn.ensemble import RandomForestClassifier
```

Model configuration:

```python
RandomForestClassifier(
    n_estimators=200,
    random_state=42
)
```

---

## Why Random Forest?

Random Forest was selected because it:

* Handles non-linear relationships
* Works well with binary risk indicators
* Is relatively resistant to overfitting
* Provides built-in feature importance metrics
* Requires minimal preprocessing
* Produces explainable outputs suitable for compliance environments

# Model Evaluation

The platform generates a standard classification report.

Implementation:

```python
classification_report(
    y_test,
    preds
)
```

Metrics produced include:

---

## Accuracy

Measures overall prediction correctness.

Formula:

```text
Correct Predictions
--------------------
Total Predictions
```

Interpretation:

Higher accuracy indicates better classification performance.

---

## Precision

Measures how many predicted high-risk countries are actually high risk.

Formula:

```text
True Positives
---------------------------
True Positives + False Positives
```

Importance:

Reduces unnecessary investigations.

---

## Recall

Measures how many actual high-risk countries were successfully identified.

Formula:

```text
True Positives
---------------------------
True Positives + False Negatives
```

Importance:

Critical in compliance settings where missing a risky jurisdiction can create regulatory exposure.

---

## F1 Score

Balances precision and recall.

Formula:

```text
2 × (Precision × Recall)
-------------------------
 Precision + Recall
```

Importance:

Useful when classes are imbalanced.

# Limitations

## Synthetic Risk Labels

The target variable is derived from engineered thresholds rather than historical compliance outcomes.

Potential impact:

* May not perfectly reflect real-world risk events.

---

## Rule-Based Thresholds

Median-based thresholds are simplistic.

Potential impact:

* Some countries may be incorrectly categorised.

---

## Limited External Signals

Current model does not include:

* News sentiment
* Geopolitical events
* Trade sanctions updates
* Financial crime data

---

## No Temporal Modelling

The current implementation analyses a single snapshot in time.

Potential impact:

* Cannot identify risk trends.
